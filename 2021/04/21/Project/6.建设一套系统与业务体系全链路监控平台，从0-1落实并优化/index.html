(2018/09–2018/10）建设一套系统与业务体系全链路监控平台，从0-1落实并优化

2021-04-21

√ 所在公司：达飞控股

√ 项目背景：

  1. 系统监控不够标准化，业务监控平台是个盲区，应用状态未知。
  2. 监控作为运维的眼睛，稳定性的必要保障，必须要做好，做全面，做稳定，所有
    服务都挂了，监控不能挂。

√ 项目人员：

研发与QA全部由我自己一个人去推进。

√ 监控方面：

  1. 机器性能监控、基础环境监控、日志监控、网络监控、访问质量监控、业务监控。

√ 监控方案：

  1. 监控系统的由来
  2. 基本组件介绍
  3. 系统架构介绍
  4. 实战规划落地

√ 监控需求：

  1. 定时执行SQL触发告警
  2. 查看主机的状态
  3. 统计接口调用耗时、QPS、成功率
  4. 分布式日志收集
  5. 接口调用链分析

√ 历史解决方案的缺点：

  1. 新增告警需要编写代码
  2. 相同功能重复开发
  3. 没有告警收敛
  4. 前端参与可视化界面的开发
  5. 使用关系型数据库存储，查询效率较低
  6. 没有JVM、中间件、告警等监控

√ 新方案的优点：

  1. 不需要自己编写代码
  2. 不同的小组开发人员可以共用同一套系统
  3. 自带告警收敛、静默功能
  4. 新增或修改告警规则很方便
  5. 自带可视化界面
  6. 使用时间序列数据库
  7. 根据监控数据去优化服务

√ 基本组件:

（系统层面-Zabbix）（业务层面-prometheus+consul+skye+Grafana）
（实现全链路监控采用开源二次开发：pinpoint，Skywalking）
